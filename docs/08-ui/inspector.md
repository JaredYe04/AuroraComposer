## 1. Background

The Event Inspector is Aurora Composer's **explainability panel** — the primary UI surface for Philosophy Principle 2: *Everything Is Explainable*. When a user clicks any note in the Piano Roll (or Timeline event marker), the Inspector displays the **full provenance chain** from user parameters through pipeline stages, rules, search steps, to the final event.

### 1.1 Canonical Display Requirement

From [philosophy.md](../00-overview/philosophy.md):

> Clicking any note in the inspector must display this chain.

**Primary summary line (required):**

```text
Generated by: Harmony Rule #42, Score: +13, Reason: Passing Tone
```

This line appears at the top of the Inspector for every selected generated event.

### 1.2 Scope

- `EventInspector.vue` component
- Provenance chain rendering
- Rule detail expansion
- Search context display
- Parameter snapshot viewer
- Parent chain navigation (repair, transform, manual edit)
- Empty and multi-select states

## 3. Academic / Theoretical Foundation

### 3.1 Provenance Chain Model (AST)

From [ast.md](../02-music-model/ast.md):

```text
Event (Note)
  └── Reason          ("Passing tone between C and E")
        └── Rule      (HarmonyRule #42)
              └── Score (+13)
                    └── History (search step, beam rank)
```

Inspector renders this tree bottom-up (event → root).

### 3.2 Explainable AI Presentation (Samek et al.)

Effective XAI UIs provide:

1. **Global summary** — one sentence (required primary line)
2. **Local explanation** — contributing factors (rule list)
3. **Context** — input state (parameters, search)

## 6. Recommended Solution

**Collapsible tree Inspector** with fixed primary summary header, sections for Rules, Search, Parameters, and Parent Chain.

Data from Tauri commands: `get_event_provenance`, `get_provenance_chain`, `get_rule_details`.

## 8. Data Structures

### 8.1 ProvenanceChain (IPC)

```typescript
interface ProvenanceChain {
  event_id: string;
  event_summary: EventSummary;
  entries: ProvenanceChainEntry[];
}

interface EventSummary {
  kind: 'Note' | 'Chord' | 'Rest' | 'Marker';
  pitch_display?: string;      // "E4"
  duration_display?: string;   // "Quarter"
  voice_name: string;
  measure_global: number;
  beat_offset: BeatOffset;
}

interface ProvenanceChainEntry {
  provenance: Provenance;
  rules: RuleDefinition[];
  display_summary: string;
  depth: number;               // 0 = immediate, 1+ = parent chain
}

interface RuleDefinition {
  id: string;                  // "harmony.allow_passing_tones"
  display_id: string;          // "Harmony Rule #42"
  name: string;
  category: string;
  description: string;
  weight: number;
  contribution_score?: number;
}

interface Provenance {
  source: ProvenanceSource;
  stage?: PipelineStageId;
  rule_ids: string[];
  eval_score?: number;
  search?: SearchContext;
  parent?: { node_id: string; patch_id?: string };
  created_at: string;
  agent: ProvenanceAgent;
  explanation?: string;
}
```

### 8.2 Inspector Store

```typescript
interface InspectorState {
  chain: ProvenanceChain | null;
  loading: boolean;
  error: string | null;
  expandedSections: Set<string>;
  selectedRuleId: string | null;
}
```

## 9. Algorithms

### 9.1 Load Chain on Selection

```text
function onSelectionChange(eventId):
    if !eventId:
        inspector.clear()
        return
    inspector.loading = true
    chain = await get_provenance_chain(projectId, eventId)
    inspector.chain = chain
    inspector.loading = false
```

### 9.2 Format Primary Summary

```text
function formatPrimarySummary(entry: ProvenanceChainEntry):
    // Backend provides display_summary; frontend may refine:
    rule = entry.rules[0]
    ruleLabel = rule ? rule.display_id : "Unknown"
    score = entry.provenance.eval_score
    scoreStr = score != null ? f"{score:+.0f}" : "—"
    reason = entry.provenance.explanation ?? "No explanation"
    return {
        line1: f"Generated by: {ruleLabel}",
        score: f"Score: {scoreStr}",
        reason: f"Reason: {reason}"
    }
```

**Example output:**

```text
Generated by: Harmony Rule #42
Score: +13
Reason: Passing Tone
```

Combined single-line variant (hover tooltip):

```text
Generated by: Harmony Rule #42, Score: +13, Reason: Passing Tone
```

### 9.3 Parent Navigation

```text
function navigateToParent(parentNodeId):
    selectionStore.selectEvent(parentNodeId)
    pianoRoll.scrollToEvent(parentNodeId)
    // triggers chain reload for parent
```

### 9.4 Multi-Select Aggregation

```text
function aggregateChains(eventIds[]):
    if eventIds.length == 0: return empty
    if eventIds.length == 1: return full chain
    chains = parallel map get_provenance_chain
    return {
        count: eventIds.length,
        dominant_stage: mode(chains.map(c => c.entries[0].provenance.stage)),
        rule_frequency: count rules across chains,
        tabIndex: 0  // switch between individual chains
    }
```

### 10.3 RulesSection

Each rule card shows:

| Field | Source |
|-------|--------|
| Display ID | `RuleDefinition.display_id` |
| Name | `RuleDefinition.name` |
| Contribution | `RuleDefinition.contribution_score` |
| Description | Expandable `RuleDefinition.description` |
| Weight | `RuleDefinition.weight` |

Click rule → highlight related notes in Piano Roll (future: `filter_by_rule`).

### 10.4 SearchSection

Displayed when `provenance.search` present:

```text
Pipeline Stage: Melody
Search Step: 42 of 64
Beam Rank: 2 / 8
Accumulated Score: 127.5
```

### 10.5 ParametersSection

Subset of `parameters_used` from composition metadata filtered by `provenance.stage`:

```text
melody.passing_tone_bias: 0.10
melody.chord_tone_bias: 0.70
search.beam_width: 8
```

### 10.6 Agent Display by Source

| ProvenanceAgent | Display |
|-----------------|---------|
| `Engine { stage }` | "Melody Engine" |
| `Plugin { plugin_id }` | Plugin display name from registry |
| `User { }` | "Manual Edit" |
| `Import { format }` | "Imported from MIDI" |

### 10.7 Empty States

| State | Message |
|-------|---------|
| No selection | "Select a note to inspect provenance" |
| Loading | Skeleton shimmer |
| Rest/Marker | "Provenance available" (may be sparse) |
| Import without chain | "Imported event — no generation history" |

## 11. Parameter Mappings

Inspector displays parameters **at generation time** (from `CompositionMetadata.parameters_used`), not live UI values. Diff indicator if current parameters differ:

```text
⚠ Parameters changed since generation. Regenerate to apply.
```

| Inspector Section | Parameter Source |
|-------------------|------------------|
| Parameters at Generation | `parameters_used` snapshot |
| Rule weights | Derived from parameters via ParamMapper |

## 12. Explainability Model

### 12.1 Full Chain Example

User clicks passing tone E4 in measure 4:

```text
┌─ Event: E4 quarter note, Melody, m.4 beat 2
│
├─ Reason: Passing tone between C4 (beat 1) and G4 (beat 3)
│
├─ Rules Applied:
│    Harmony Rule #42 "Allow passing tones on weak beats"  +13
│    Melody Rule #17 "Prefer stepwise motion"             +4
│
├─ Search: Melody stage, step 42, beam rank 2/8, score 127.5
│
├─ Agent: Melody Engine
│
└─ Parameters: passing_tone_bias=0.10, beam_width=8
```

### 12.2 Manual Edit Chain

User changed C4 → D4:

```text
Generated by: Manual Edit
Reason: User changed pitch C4 → D4
Parent: → [Click to view original C4 event]
```

```text
Generated by: Jazz Standard Harmony Plugin
Rule: harmony.jazz-ii-v-i  +18

Agent: Plugin (com.aurora.plugins.jazz-standard)
```
